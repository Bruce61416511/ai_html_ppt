Category 02 · 质量意识

差的 RAG vs 好的 RAG — 差在哪？

⚠️ 差的 RAG 什么样

答非所问

问了"变压器耐压标准"，回了"变压器运输规范"——检索没命中

幻觉编造

文档里明明没有，AI 自己编了一个答案——Prompt 没约束好

搜不到关键信息

扫了 50 页 PDF，最有用的那段在第 42 页——切片太大漏掉了

✅ 好的 RAG 怎么做

文档预处理 — Garbage In, Garbage Out

扫描件必须 OCR。表格必须结构化。否则 RAG 就是个瞎子

智能切片 — 粒度决定精度

按语义段落切，不是按字数硬切。相邻片段保留 10% 重叠防截断

混合检索 + 严谨 Prompt

语义 + 关键词双路召回，Prompt 约束"只根据提供的文档回答，不知道就说不知道"

关键认知：RAG 不是"搭好就完"——文档质量、切片策略、检索精度、Prompt 约束，任一环节掉链子，效果就是天壤之别

合并原 slide 10+11 的精华，但不列技术清单。左右对比让听众秒懂：差在哪、好怎么做。Embedding、BM25、Reranker、RAGAS 这些术语全部藏在 speaker notes 里。

Category 02 . 多模态

🖼️ 图片 / 视频 的 RAG 如何制作？

传统 RAG 只处理文字，多模态 RAG 让图片、视频也可检索和问答

📷
图片 → 三种理解路径

OCR（光学字符识别）：扫描件、证照等直接提取文字存储。
CLIP 向量嵌入：产品图、设计图生成向量，支持“以图搜图”“文字搜图”。
多模态 LLM：GPT-4o / Qwen-VL 深度理解图片内容，支持看图问答与自动标注。

🎬
视频 → 抽帧（图片）+ 语音转写（文本）+ 时间戳

每 2-3 秒抽一帧→走图片管线（OCR/CLIP/多模态 LLM）。
同时 Whisper 转写音频为文字。
时间戳将帧与文字绑定，检索时可精确跳转到视频对应秒数。

🔄
依赖大模型实际流程

图片/视频 → 多模态LLM理解 → 切片存储 → 用户提问 → 混合检索 → 多模态 LLM 生成回答 → 回答中引用图片/视频片段

多模态 RAG 扩展了传统 RAG 的能力边界。关键技术：OCR（光学字符识别）、CLIP多模态嵌入、Whisper语音转写、多模态LLM。适用场景：产品手册中的图表检索、培训视频知识库、设备故障照片诊断等。

Category 03

📈 预测分析与库存决策

两条路径，一个范式：商贸与装备制造各自的预测 → 决策 → 智能体闭环

📊 商贸：销量预测 → 库存决策 → 智能体

销量预测

四算法分层：TFT / DeepAR → 爆款 · LightGBM → 腰部 · Croston → 长尾
融合促销、天气、节假日、社媒热度等多因素

库存决策

安全库存（Monte Carlo 仿真）· 多仓调拨（整数规划）· 滞销预警（规则引擎 + 三级预警）

智能体

LLM 调度中枢：自然语言提问 → 意图识别 → 调用算法 → 人话输出

⚡ 装备制造：物料 → 备件 → 呆滞 → 智能体

BOM 物料需求预测

MRP 反向推导 + 时序预测：根据生产订单交付日倒推进料计划

备件库存 & 呆滞预警

泊松分布 / Weibull 分布（备件）· 规则引擎（呆滞：库龄+排产状态+通用性 → 三级预警）

智能体

LLM 调度层与商贸完全相同，底层算法切换为 MRP + 统计模型

共同范式：LLM 负责理解意图、输出人话；算法负责预测 / 仿真 / 优化 — 各司其职，调度层复用

两条路径覆盖 Category 03 全貌：商贸路径（预测 → 决策 → 智能体），装备制造路径（物料 → 备件 → 呆滞 → 智能体）。底层算法按场景替换，LLM 调度层完全复用。

Category 03 · 分层预测

SKU 销量预测 · 四算法卡片

日均销量分层（爆款 >100件/天 | 腰部 10-100 | 长尾 <10）→ 每层对号入座一个算法

🔴 TFT — 爆款

① 输入

历史日销量（365天+）+ 促销日历 & 折扣深度 + 天气（温度/降雨）+ 节假日 & 大促日 + 社媒搜索指数

② 算法

Temporal Fusion Transformer（Google/Oxford 2021）
多因素时序模型，自带变量重要性归因

⚡ 零代码：AWS Forecast | Google Vertex AI | 阿里云 PAI

③ 输出

未来 30 天日销量预测
涨跌归因：促销贡献多少、天气贡献多少

🟠 DeepAR — 爆款

① 输入

历史日销量（180天+）+ 促销标签 + 产品属性（价格带 / 类目 / 品牌）

② 算法

DeepAR（Amazon 2019）
自回归循环网络，支持跨产品学习

⚡ 零代码：AWS Forecast（DeepAR 为 Amazon 原生算法）

③ 输出

未来 30 天销量概率区间（P10 / P50 / P90）
新品（无历史销量）可从相似老品迁移冷启动

🟡 LightGBM — 腰部

① 输入

历史日销量（90天+）+ 是否参与促销 + 星期几特征（周末效应）+ 售价变动记录

② 算法

LightGBM（Microsoft 2017）
梯度提升决策树，快·轻·CPU 可跑

⚡ 零代码：阿里云 PAI | Azure AutoML | Google Vertex AI

③ 输出

未来 30 天日销量预测

🟢 Croston — 长尾

① 输入

每次卖出日期 + 每次卖出件数 + 上次卖出距今多少天

② 算法

Croston 间歇需求模型（Croston 1972）
不看日销量，只看卖出事件

⚡ 零代码：无主流云厂商内置，需自写（逻辑简单，几十行 Python）

③ 输出

实时监测数据 + 国家饮用水标准 GB5749 + 地方水务管理条例

② 方法

规则引擎：逐项对照 GB5749 限值，自动判定达标/超标
多站点联动：上游异常时，自动检查下游站点是否受影响

③ 输出

合规达标报告（可直接提交监管部门）+ 超标项目红色高亮 + 历史趋势图

🏭 延伸：安全生产环境监控

① 输入

生产环境传感器（温度、湿度、粉尘浓度、噪音、有毒气体）+ 安全生产标准（GBZ 2.1）+ 历史事故数据

② 方法

同时序异常检测 + 规则引擎，复用水质监控的技术栈
额外：视频分析（工人未戴安全帽 / 危险区域入侵）

③ 输出

安全风险实时预警 + 工伤风险评估 + 整改建议

水质监控的核心挑战：传统人工抽检频率低、漏检风险高、报告编写耗时。AI 方案做到 24/7 全量监控，实时告警，报告自动生成。同一技术栈可复用到安全生产环境监控。

Category 06 · 医疗

🏥 处方合规审查 · 规则引擎 + LLM 双重审查

规则引擎做硬约束，LLM 做语义理解，两层过滤确保处方安全

🧪 第一层：规则引擎硬约束

① 输入

处方单（药品名称、剂量、用法、疗程）+ 药典配伍禁忌库

② 规则

"十八反""十九畏"传统配伍禁忌 + 现代药物相互作用数据库
剂量上下限校验（成人/儿童/孕妇分组）
疗程合理性校验（某些药物不能超过 N 天连续服用）

③ 输出

硬性违规标注（❌ 禁忌配伍 / 超剂量 / 疗程超限）+ 具体条目及依据

🧠 第二层：LLM 语义理解

① 输入

处方单 + 患者病史（诊断、过敏史、当前用药清单）+ 临床指南

② 方法

LLM 理解处方用药逻辑：诊断与用药是否匹配、联合用药是否重复、是否存在更优选择
对照临床指南判断用药方案的合理性

③ 输出

软性风险标注（⚠️ 建议复查 / 注意监测）+ 修改建议 + 替代方案推荐

📋 最终输出：处方审查报告

两层过滤后形成结构化报告：总体评级（✅合规 / ⚠️有风险 / ❌违规）+ 硬性违规条目（必须修改）+ 软性风险条目（建议复查）+ 风险条目高亮与修改建议

处方审查的关键：规则引擎处理硬性禁忌（十八反、剂量超限），LLM 处理软性风险（用药逻辑、方案优化）。两层分工，互为补充，确保既不漏报硬性违规，也能发现潜在风险。

Category 06 · 专业服务 · 数据脱敏

🤝 合同风险审查 · 🔒 数据脱敏与隐私合规

合同审查：LLM 条款解析 + 法规对比 → 风险评分　｜　数据脱敏：PII 自动识别 → 动态脱敏 → 合规报告

📜 合同条款解析与风险识别

① 输入

合同文本（PDF / Word）+ 法律法规库（民法典、合同法、行业条例、司法解释）+ 历史合同风险案例库

② 方法

LLM 条款级解析：提取付款条款、违约条款、知识产权条款、争议解决条款等关键条款
与法规库对比 + 历史案例匹配 → 风险等级判定（高/中/低）
⚡ 零代码：阿里云 合同审查 | 百度智能合同 | e签宝

③ 输出

风险条款高亮 + 修改建议 + 合规评分（0-100）+ 争议解决偏好分析（仲裁/诉讼/协商）

🔒 数据脱敏与隐私合规

① 输入

业务数据库 / 文档 / 日志文件 + 个人信息保护法 + GDPR / 行业合规要求

② 方法

NER 模型自动识别 PII：姓名、身份证号、电话、地址、银行卡、生物特征
动态脱敏（按角色可见）/ 静态脱敏（不可逆）
⚡ 零代码：AWS Macie | 阿里云 数据安全中心 | Privitar

③ 输出

脱敏后数据 + 合规报告（已脱敏字段清单、脱敏方式、残留风险评估）

合同审查的价值：传统法务审查一份合同需 2-4 小时，AI 将其缩短到分钟级，同时覆盖更全面的法规库。数据脱敏是跨行业刚需，尤其在医疗、金融、专业服务领域。
