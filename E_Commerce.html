# 产品概念生成 · 全流程 Prompt

## 角色

你是电商产品策略顾问。基于多源数据，完成从市场信号捕捉、用户需求挖掘到跨界概念生成的全流程分析，最终输出可落地的产品开发建议。

三步串联：第一步的品类结果 → 作为第二步、第三步的分析范围，确保三步围绕同一批品类。

## 第一步：看信号 — 多源数据采集 & 趋势发现

### 数据采集

- 🔧 八爪鱼 → 平台热销榜排名、1688 搜索热词

- 🔧 蝉妈妈 → 抖音/TikTok 搜索趋势、爆款数据

- 🔧 Apify → Amazon 搜索趋势、Shopee 热销、Instagram 标签

- 🔧 Google Trends → 搜索量趋势对比、地域分布、季节性规律

### AI 分析

用 🧠 GPT-4o 完成：

1. 多源爬取 + 清洗标准化：将不同来源的数据统一为结构化格式，去除重复和噪声

2. 识别上升品类 + 增速排序：找出搜索量、销量、讨论量同步增长的品类，按增速降序排列 TOP 10

3. 判断趋势真假：区分短期炒作（突然爆量后回落）和长期趋势（稳定增长 3 个月以上），标注每个品类的趋势可信度

### 输出要求

- 搜索量趋势（月环比增长率）

- 转化率（搜索到购买的转化比例）

- 评论热度（评论数增速 + 情感倾向）

- 增速排名 TOP 10 + 趋势真假判断

- 3 个最值得切入的品类，每个附 100 字入场理由

→ 以上 TOP 3 品类作为第二步、第三步的目标品类，后续分析围绕这 3 个品类展开

## 第二步：挖痛点 — 差评痛点挖掘

### 数据采集

以第一步输出的 TOP 3 品类为目标，采集：

- 🔧 八爪鱼 → 各平台商品 1-2 星差评、问答讨论

- 🔧 魔镜市场情报 → 竞品口碑数据、客服投诉记录

- 🔧 Apify → Reddit / Trustpilot 用户吐槽、海外独立站评价

- 🔧 平台后台 → 退款退货原因、售后纠纷高频词

### AI 分析

用 🧠 GPT-4o 完成：

1. 提取差评高频词：按出现频次统计 TOP 20 负面关键词（如“太重”、“续航短”、“掉色”）

2. 聚类归纳痛点：将高频词归纳为 5-8 个核心需求缺口，区分功能性痛点（产品不好用）和体验性痛点（用起来不爽）

3. 排序 + 情景模拟：按影响面（提及人数 × 严重程度）排序，模拟改进后的用户满意度变化

### 输出要求

- 5-8 个核心需求缺口，每个标注频次、影响面、痛点类型

- 每个缺口给出具体的产品改进方向（100 字内）

- 痛点 × 改进方案矩阵，标注优先级（高/中/低）

# 竞品价格分析 Prompt

你是一个电商竞品分析师。请基于提供的竞品价格数据，完成以下分析：

【数据采集工具】
- Keepa：追踪 Amazon 商品历史价格，每日自动抓取标价、券后价、促销标签
- 魔镜市场情报：监控竞品价格变动、促销活动节奏
- 零代码替代：AWS Forecast | Google Vertex AI | Azure AutoML | 阿里云 PAI | 华为云 ModelArts
  （无需编写 Prophet/LSTM 代码，上传历史价格数据即可自动建模预测）

【输入数据】
- 竞品 X 近 7 天每日标价、券后价、促销标签（满减/秒杀/会员价）
- 竞品 X 近 30 天价格历史（用于对比趋势）
- 平台大促日历（618/双11/品类日等节点标注）

【分析任务】
1. 价格波动识别：标注 7 天内所有调价节点，计算幅度、频率、方向
2. 策略模式判定：高频调价/阶梯降价/活动前涨价/跟随定价/清仓甩卖
3. 营销意图推测：冲销量/清库存/拉新客/利润收割/防守阻击
4. 应对建议：针对每种意图给出 3 条可执行对策，标注优先级

【AI 模型】
- Prophet（Meta 开源）：分解趋势 + 季节性 + 节假日效应，可解释性强
- LSTM（深度学习）：捕捉非线性长程依赖，精度更高
- 两者互补：Prophet 做基线预测，LSTM 做精细校准

【输出格式】 Markdown 表格：时间 | 动作 | 模式 | 意图 | 我方对策 | 优先级
末尾附 200 字结论摘要，给出「跟进 / 观望 / 反击」建议。

03 · 智能选品决策

四维选品矩阵：从"感觉"到"科学"

💡 利润维度

流程：📥 标价/券后价（← Keepa + 魔镜市场情报 每日抓取） + 我方预估成本（采购/物流/佣金）→ 📊 XGBoost 预估利润率 → 📊 利润得分
📊 XGBoost：开源集成学习算法，AWS SageMaker / Google Vertex AI / Azure ML / 阿里云 PAI 均封装为零代码服务，上传 CSV 即可预测利润率

💡 需求维度

流程：📥 搜索量 / 评论增速 / 趋势加速度（← 蝉妈妈 / Apify / Google Trends）→ 📊 加权评分公式 → 📊 需求得分
📊 加权评分公式：多指标加权评分，需求得分 = 搜索量分 × 0.5 + 评论增速分 × 0.3 + 趋势加速度分 × 0.2

💡 竞争强度

流程：📥 竞品数量 / 价格带饱和度 / 评论壁垒（← 八爪鱼 + Keepa 多源聚合）→ 📊 加权评分（反比） → 📊 竞争得分
📊 加权评分（反比）：多指标加权评分（反比），竞争得分 = 数量分 × 0.4 + 饱和分 × 0.3 + 壁垒分 × 0.3，分越高 = 竞争越弱

💡 供应链风险

流程：📥 供应商数量 / 交期 / 起订量（← 1688 + 行业数据）→ 📊 加权评分 → 📊 供应链得分
📊 加权评分：多指标加权评分，供应链得分 = 供应商数量分 × 0.4 + 交期分 × 0.3 + 起订量分 × 0.3，分越高 = 越稳

综合决策：📋 四维综合排序 → TOP N 选品清单 + 理由 → 人审核拍板

# 内容模板提炼 · Prompt

你是一个内容策略分析师。请基于输入的视频（或文字/图片素材），完成模板提炼与入库：

【输入】
- 一个高转化率的视频（提供口播文本 + 分镜描述 + 关键帧截图）
- 该视频的历史效果数据（播放量、完播率、转化率、互动率）

【分析任务】
1. 结构拆解：识别视频的叙事节奏
   - 开篇钩子：前 3 秒用了什么手法（痛点/悬念/反常识/强视觉）？
   - 产品展示：如何演示核心卖点（对比/场景化/数据可视化）？
   - 说服构建：用了什么信任元素（认证/好评/专家/KOL）？
   - 转化引导：CTA 的位置、话术、紧迫感策略

2. 模板提炼：将以上分析抽象为可复用的模板
   - 脚本框架：每段的时长占比、话术模式、情绪曲线
   - 视觉模板：构图规律、配色偏好、字幕风格
   - 标注适用平台及平台偏好（Amazon / TikTok / 小红书 / Shopee）

3. 向量化入库
   - 模板输出为结构化 JSON（含框架 + 标签 + 平台偏好 + 效果基线）
   - 生成文本 embedding（OpenAI text-embedding-3-small），写入向量数据库
   - 标签体系：品类、风格、平台、时长、转化率区间

JSON，每个字段含具体可执行的模板内容：

第三部分：供应链。在电商里，库存就是现金流。卖得好的品缺货是灾难，卖不动的品积压也是灾难。AI的核心价值就是让预测更准、补货更聪明。

01 · 需求预测引擎

多因素融合，让预测不再是"拍脑袋"

💡 TFT · 多因素预测模型

① 根据：Google/Oxford 2021 论文，M5 时序预测竞赛前列，自带可解释性

② 输入

历史销量（店铺后台）+ 促销标签/折扣深度（商家后台活动配置）+ 城市天气（和风天气/OpenWeatherMap API）+ 节假日/大促日（人工维护大促日历表）+ 社媒搜索指数（Google Trends / 蝉妈妈 / 百度指数）

③ 工具

TFT 算法（Temporal Fusion Transformer）

⚡ 零代码：AWS Forecast | Google Vertex AI | 阿里云 PAI

④ 输出

未来30天日级销量预测表

📊 分层预测 · 量级分策略

① 根据：零售行业通行做法，不同量级商品统计特性不同，需分层建模

② 输入

商品日均销量分层（店铺后台订单数据）：爆款（>100件/天）、腰部（10-100）、长尾（<10）

③ 工具

爆款：DeepAR | 腰部：LightGBM | 长尾：Croston 间歇加权评分公式

⚡ 零代码：DeepAR → AWS Forecast  |  LightGBM → 阿里云 PAI / Azure AutoML  |  Croston → 需自写

均为专用时序预测算法，非语言大模型

④ 输出

爆款未来30天销量预测 / 腰部未来30天销量预测 / 长尾未来30天销量预测

⚡ TFT · 算力 & 成本

算力：高 — 全量 SKU 训练，需 GPU
成本：贵 — 按训练时长 + 调用次数计费
适合：SKU 少、数据丰富、预算充足

💰 分层 · 算力 & 成本

算力：低 — 仅爆款用 DeepAR，其余轻量
成本：省 — 总成本约 TFT 的 20-30%
适合：SKU 多、长尾占比大、控预算

左边TFT是单品精细预测，右边分层是全店策略。爆款用TFT日级精算，腰部用LightGBM周级够用，长尾用Croston判断会不会卖。好钢用在刀刃上——全店1000个SKU不用都上TFT。——它能同时处理销量、促销、天气、节假日等多种因素，输出未来30天的预测和置信区间。分层预测的关键：爆款要日粒度精细管理，长尾商品用间歇加权评分公式，不要一刀切。

02 · 智能库存管理

安全库存 + 多仓调拨 + 滞销预警

📦 安全库存动态计算

① 输入

需求预测（上一页 TFT 输出）+ 供应商历史交货天数 + 目标服务水平 98%

② 模型

Monte Carlo 仿真：同时随机抽取历史需求 + 历史交货天数，仿真一万次 → 计算不同库存水平下的断货概率
⚡ @RISK / Oracle Crystal Ball（零代码）

③ 输出

98% 不断货所需的最小安全库存量

🚚 多仓智能调拨

① 输入

各仓当前库存 + 各区域需求预测（← 上一页 TFT 输出）+ 仓间运输成本

② 模型

整数规划（Google OR-Tools）求解：运输成本 + 持有成本 + 缺货损失最小化
⚡ Gurobi / CPLEX（企业版）

③ 输出

最优调拨方案：从哪仓调到哪仓、调多少

🚨 滞销预警

① 输入

商品库龄（入仓日期）+ 近30天销量趋势（店铺后台）+ 品类利润率（渠道毛利率）

② 模型

规则引擎综合判断：库龄(30/60/90天) + 销量趋势(是否还在走量) + 利润率(薄利/厚利) → 黄/橙/红三级预警
⚡ 旺店通 / 聚水潭（ERP 内置）

③ 输出

预警等级
